# API Contracts

This directory contains API contract specifications that define the complete interface between the service and its clients.

## Purpose

API contracts provide a formal specification of:
- Endpoints and their behaviors
- Request/response formats
- Authentication and authorization requirements
- Error handling patterns
- Versioning strategies

## Files

- **`api-contract.md`** - Complete API contract specification for the FastAPI Service, including all endpoints, authentication, response formats, and error handling.

## Usage

These contracts serve as:
- **Documentation** for API consumers
- **Reference** for frontend developers
- **Specification** for API client generation
- **Validation** criteria for API testing
