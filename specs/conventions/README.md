# Coding Conventions

This directory contains coding conventions, standards, and best practices for the FastAPI Service project.

## Purpose

These conventions ensure consistency across the codebase and provide guidelines for:
- Code style and formatting
- Architecture patterns
- API design standards
- Testing practices
- Security guidelines

## Files

- **`00-repository-overview.md`** - Repository overview, tech stack, and architecture patterns
- **`01-python-conventions.md`** - Python coding conventions and style guidelines
- **`02-api-design.md`** - API design standards and best practices
- **`03-module-structure.md`** - Module structure and organization patterns
- **`04-testing-standards.md`** - Testing conventions and best practices
- **`05-security-guidelines.md`** - Security best practices and guidelines

## Usage

These conventions should be:
- **Followed** by all developers working on the project
- **Referenced** during code reviews
- **Updated** as the project evolves and new patterns emerge
- **Enforced** through linting tools and pre-commit hooks
