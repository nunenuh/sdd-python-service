# Documentation Directory Structure

This directory contains comprehensive documentation for the FastAPI Service boilerplate.

## Directory Structure

- **architecture/** - System architecture diagrams and design documents
- **archive/** - Archived documentation and historical references
- **current/** - Current project status and active documentation
- **infra/** - Infrastructure documentation (Docker, deployment, etc.)
- **issues/** - Known issues and bug tracking documentation
- **implementation/** - Implementation details and technical specifications
- **planning/** - Project planning documents and roadmaps
- **troubleshooting/** - Troubleshooting guides and common solutions
- **quickstart/** - Quick start guides and getting started documentation
- **api/** - API documentation and endpoint references
- **deployment/** - Deployment guides and procedures
- **development/** - Development setup and contribution guidelines
- **testing/** - Testing documentation and test strategies

## Documentation Standards

- Use Markdown format (.md) for all documentation
- Include code examples where applicable
- Keep documentation up to date with code changes
- Use clear headings and structure
- Add diagrams for complex concepts

## Contributing

When adding new documentation:

1. Place files in the appropriate directory
2. Update this README if adding new sections
3. Follow the existing documentation style
4. Include examples and code snippets
5. Keep documentation concise and focused
